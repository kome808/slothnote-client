## 一句話摘要
從「憑感覺寫程式」(Vibe Coding) 到「自主代理開發」(Agentic Coding)，工程師的角色已轉向定義目標與監督 AI 團隊。

## 關鍵重點
1. **演進三階段**：從 Autocomplete (輔助打字) 到 Chat (對話生成碼) 到 Agentic (自主規劃與修復)。
2. **IC 變 Manager**：人類不再是寫 Code 的人，而是負責「推理 → 行動 → 觀察 → 審查」循環中的決策者。
3. **工作流資產化**：知識不再只在腦中，而是轉化為專案中的 `CLAUDE.md`、Skills 或 Prompts，實現版本化與持續升級。

## 原文核心觀點
Agentic Coding 的質變在於「自主性」。透過多 Agent 協作（前端、後端、測試、文件），開發效率大幅提升。這不僅改變了寫程式的方法，更是一種全新的管理邏輯：設定目標 → Agent 執行 → 人類審查。

## AI 洞察
> 這篇文章提到的「IC 變 Manager」正好呼應了 Anthropic 的研究（見 [ai-coding-skills](./2026-02-20-ai-coding-skills.md)）。當我們從「執行者」轉向「管理者」時，雖然產出效率提升，但也面臨「失去底層掌握力」的風險。這意味著未來的資深工程師，具備「高階審核（Code Review）」與「系統架構」的能力將比單純寫語法更為重要。
